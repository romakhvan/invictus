# Список мобильных тестов для запуска
# Запуск: python run_tests.py  (MODE = "mobile" в run_tests.py)
#
# Формат: путь к тесту | флаги (необязательны, дополняют PYTEST_ARGS)
#   tests\mobile\bookings\test_foo.py | --mobile-no-reset -k bar --keepalive
#
# Директивы (управляют поведением скрипта):
#   # ALLURE: on|off         — генерировать Allure-отчёт
#   # OPEN_REPORT: on|off    — открыть отчёт после завершения (только если ALLURE: on)
#   # INTERACTIVE: on|off    — добавляет -s ко всем тестам (нужно если есть input())
#   # PYTEST_ARGS: -v -s     — базовые аргументы для всех тестов в файле
#
# Флаги pytest:
#   -v                  подробный вывод (имя каждого теста)
#   -m mobile           запускать только @pytest.mark.mobile тесты
#   -s                  показывать print() из тестов
#   -k "doctors"        фильтр по имени теста (подстрока или выражение)
#
# Кастомные флаги Appium:
#   --mobile-no-reset   не сбрасывать данные приложения между тестами
#   --keepalive         показать интерактивное меню отладки после теста (оставляет приложение открытым)
#   --mobile-ui-logs    логи UI-слоя (WAIT/CLICK/SWIPE/etc.)
#   --onboarding-phone  конкретный номер телефона для теста онбординга (если не указан — ищется автоматически)
