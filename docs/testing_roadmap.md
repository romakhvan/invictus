## Следующие шаги

### 1. Настройка окружения (ПРИОРИТЕТ: ВЫСОКИЙ)

#### 1.1. Установка зависимостей
```bash
# Установить Python пакеты
pip install -r requirements.txt

# Установить браузеры для Playwright
playwright install chromium
```

#### 1.2. Настройка конфигурации
Отредактировать `src/config/app_config.py`:
- Указать реальный `WEB_BASE_URL`
- Настроить параметры Appium (если нужно)
- Указать путь к мобильному приложению

#### 1.3. Проверка подключений
- Убедиться, что MongoDB доступна
- Проверить доступность веб-сайта
- Запустить Appium сервер (если нужны мобильные тесты)

### 2. Создание тестовых данных и утилит

#### 2.1. Тестовые пользователи
Создать `src/utils/test_data.py`:
- Тестовые учетные данные
- Генераторы тестовых данных
- Хелперы для очистки данных

#### 2.2. Утилиты для UI
Создать `src/utils/ui_helpers.py`:
- Ожидания элементов
- Скриншоты при ошибках
- Логирование действий

### 3. Создание Page Objects для реальных страниц

#### 3.1. Определить ключевые страницы/экраны
Начать с самых важных:
- Страница входа (Login)
- Главная страница (Home/Dashboard)
- Профиль пользователя
- И т.д.

#### 3.2. Создать Page Objects
Для каждой страницы:
- Определить селекторы
- Реализовать методы взаимодействия
- Добавить проверки загрузки

### 4. Написание первых тестов

#### 4.1. Smoke тесты
Начать с простых проверок:
- Загрузка главной страницы
- Базовая навигация
- Проверка ключевых элементов

#### 4.2. Интеграционные тесты
Комбинировать UI и Backend:
- Создание пользователя через UI → проверка в БД
- Изменение данных в UI → проверка в БД

### 5. Организация существующих тестов

Переместить существующие backend тесты в `tests/backend/`:
- `test_coach_wallet.py`
- `test_rabbitholev2_no_duplicate_users.py`
- `test_subscription_gaps.py`
- И другие

### 6. Настройка CI/CD (опционально)

- GitHub Actions / GitLab CI
- Автоматический запуск тестов
- Отчеты о тестировании

## Рекомендуемый порядок выполнения

1. **Настройка окружения** (30 мин)
2. **Создание утилит для тестовых данных** (1-2 часа)
3. **Page Object для страницы входа** (1-2 часа)
4. **Первый реальный тест** (30 мин)
5. **Расширение покрытия** (постепенно)

## Критерии готовности

- ✅ Можно запустить тесты: `pytest -m web`
- ✅ Есть хотя бы один рабочий Page Object
- ✅ Есть хотя бы один рабочий тест
- ✅ Тесты интегрируются с существующими backend проверками
