# QA Documentation

Главная точка входа в QA-документацию проекта.

## Разделы

- [Onboarding](onboarding/README.md) - локальная настройка, запуск и базовый
  рабочий цикл.
- [Architecture](architecture/README.md) - устройство фреймворка, стратегия,
  правила поддержки и отчётность.
- [Platforms](platforms/README.md) - platform-specific документация по
  `backend`, `web` и `mobile`.
- [Domain](domain/README.md) - QA-справочники по бизнес-данным и доменным
  сущностям.
- [Legacy](legacy/README.md) - карта исторических и вспомогательных документов.

## Правило поддержки

- Новые QA-документы добавляем в один из разделов `onboarding`,
  `architecture`, `platforms`, `domain` или `legacy`.
- Корневой `docs/qa/README.md` остаётся только картой навигации, без длинных
  deep-dive разделов.
