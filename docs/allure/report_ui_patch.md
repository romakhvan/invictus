# UI-патч Allure-отчёта

Этот документ описывает локальную постобработку готового `allure-report`.

## Зачем нужен патч

Vanilla Allure не всегда удобно отображает HTML-вложения внутри iframe, особенно когда вложение высокое или содержит таблицы. В проекте поверх стандартного отчёта применяется локальный UI-патч, чтобы HTML-вложения было проще читать без ручного ресайза и прокрутки в нескольких контейнерах сразу.

## Где реализовано

Патч реализован в `src/utils/allure_report_patcher.py`.

Функция `patch_allure_report(report_dir)`:

1. создаёт `custom-styles.css`;
2. создаёт `custom-iframe-resize.js`;
3. вшивает ссылки на них в `allure-report/index.html`.

## Что меняет патч

- растягивает iframe HTML-вложений по ширине;
- увеличивает минимальную высоту просмотрщика;
- убирает лишние ограничения по высоте у промежуточных контейнеров;
- добавляет ресайз по `ResizeObserver`, чтобы iframe подстраивался под содержимое;
- улучшает читаемость модального окна вложений.

## Что патч не меняет

Патч не определяет:

- какие `feature/story/title` должны быть у теста;
- какие `allure.attach(...)` обязательны;
- какие параметры должны быть в backend-отчёте;
- как группировать данные в summary и HTML-таблицах.

Это относится к правилам репортинга и описано в `backend_reporting_rules.md`.
