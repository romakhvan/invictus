# Тесты Web-Kassa

## Описание
Тесты для проверки создания фискальных чеков (web-kassa) при проведении транзакций в клубах.

## Структура данных

### Коллекции MongoDB:
- **transactions** - транзакции с полем `webKassaIds` (массив ID чеков)
- **webkassas** - фискальные чеки со статусами и деталями
- **clubs** - информация о клубах

### Связи:
```
transactions.webKassaIds[] -> webkassas._id
transactions.clubId -> clubs._id
webkassas.club -> clubs._id
```

## Тесты

### 1. test_webkassa_status_by_clubs (МОНИТОРИНГ) ⭐
**Файл:** `tests/backend/test_webkassa_monitoring.py`

Анализирует все транзакции за последние 7 дней и показывает статистику по каждому клубу.

**Что показывает:**
- Количество транзакций с успешными чеками
- Количество транзакций с ошибочными чеками
- Количество транзакций без чеков
- До 5 примеров ошибочных транзакций (с деталями)
- До 5 примеров транзакций без чеков (с деталями)
- Топ-30 клубов с наибольшим количеством проблем

**Преимущества:**
- Не зависит от файла со списком клубов
- Показывает реальную картину по всем клубам
- Группировка по статусам чеков
- Примеры проблемных транзакций для расследования

**Запуск:**
```bash
pytest tests/backend/test_webkassa_monitoring.py::test_webkassa_status_by_clubs -v -s
```

**Результаты за 7 дней (17.12.2025):**
- Транзакций: 6,958
- Клубов: 54
- С успешными чеками: 503 (10.3%)
- С ошибочными чеками: 101 (2.1%)
- Без чеков: 4,268 (87.6%)

### 2. test_receipts_creation_for_web_kassa_clubs
**Файл:** `tests/backend/test_web_kassa_receipts.py`

Проверяет создание чеков для клубов из списка `data/clubs_with_web_kassas.txt`

**Что проверяет:**
- Загрузка списка клубов из файла
- Наличие транзакций с `webKassaIds` за последние 30 дней
- Существование всех чеков в коллекции `webkassas`
- Согласованность данных между `transactions` и `webkassas`
- Процент успешных чеков (минимум 60%)

**Параметры:**
- `days` - период проверки (по умолчанию 30 дней)
- `min_success_rate` - минимальный процент успешных чеков (по умолчанию 60%)

**Параметры:**
- `days` - период проверки (по умолчанию 30 дней)
- `min_success_rate` - минимальный процент успешных чеков (по умолчанию 60%)

**Результаты на PROD за 30 дней (17.12.2025):**
- Клубов из списка: 33
- Найдено в БД: **29 (87.9%)**
- Транзакций за 30 дней: **16,525**
- Транзакций с чеками: **3,880 (23.5%)**
- Успешных чеков: **3,502 (90.3%)**
- Чеков с ошибками: **378 (9.7%)**

### 3. test_webkassas_basic_structure
**Файл:** `tests/backend/test_web_kassa_receipts.py`

Быстрая проверка структуры коллекции `webkassas`

**Что проверяет:**

## Нормализация названий клубов

Тест использует нормализацию названий для более точного поиска:

### Базовая нормализация:
- Игнорирование регистра (Go → go, GO → go)
- Удаление лишних пробелов
- Приведение к единому формату

### Учет вариаций транслитерации казахских имен:
- `Qonaev` → `Qonayev` (e/y варианты)
- `Satpayeva` → `Satpayev` (окончания -a/-нет)
- `Sharipova` → `Sharipov` (окончания -a/-нет)
- `Mangilik El` → `Mangilik Yel` (E/Y варианты)
- `Koshkarbayeva` → `Koshkarbayev` (окончания -a/-нет)

Эти правила позволяют находить клубы независимо от:
- Написания "Invictus Go" или "Invictus GO"
- Вариантов транслитерации казахских фамилий
- Различий в окончаниях имен собственных

## Файлы данных

Список клубов, в которых должны создаваться фискальные чеки.


## База данных
Тесты работают с **PROD** базой данных (MONGO_URI_PROD).

## Примечания
- Тесты выполняются в read-only режиме
- Не создают и не изменяют данные
- Используют фикстуру `db` с подключением к PROD
